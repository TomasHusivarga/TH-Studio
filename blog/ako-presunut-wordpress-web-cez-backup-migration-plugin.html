Ako presunúť WordPress web cez Backup Migration plugin

Backup Migration je praktický plugin, keď potrebuješ presunúť celý WordPress web. Nie je to nástroj na kopírovanie jednej sekcie ako text v dokumente. Ber ho skôr ako zálohu a obnovu celého webu.

TémaWordPress presun
NástrojBackup Migration plugin
Pomoc s WordPressom

Kedy ho použiť

Použil by som ho vtedy, keď chceš presunúť celý web z jedného hostingu na druhý, spraviť pracovnú kópiu alebo obnoviť web zo zálohy. Ak chceš preniesť len jednu stránku z Gutenbergu, často stačí kopírovať blokový kód alebo použiť WordPress export/import.

Pred presunom

Pred migráciou by som si neskontroloval iba plugin. Dôležitý je aj nový hosting: verzia PHP, veľkosť webu, limit uploadu, voľné miesto, SSL certifikát a to, či už na cieľovom webe nie je obsah, ktorý sa nesmie prepísať.

Obnova cez Backup Migration vie prepísať cieľový WordPress. Preto je dobré mať zálohu aj z novej inštalácie, hlavne ak tam už niekto niečo nastavoval.

Postup krok za krokom

  1. Na pôvodnom webe aktualizuj WordPress, pluginy a tému, ak je to bezpečné.
  2. Nainštaluj plugin Backup Migration.
  3. V nastaveniach pluginu sprav novú zálohu cez tlačidlo na vytvorenie zálohy.
  4. Stiahni záložný súbor do počítača a nechaj si ho bokom.
  5. Na novom webe nainštaluj čistý WordPress a ten istý Backup Migration plugin.
  6. Nahraj zálohu do pluginu a spusti obnovu podľa pokynov pluginu.
  7. Po obnove sa znovu prihlás do administrácie, často už s údajmi z pôvodného webu.
  8. Skontroluj URL adresy, menu, formuláre, obrázky, cache, SSL a responzívne zobrazenie.

Keď ide len o jednu stránku

Backup Migration nie je najlepší nástroj, ak chceš presunúť iba jednu podstránku. Vtedy často stačí skopírovať Gutenberg bloky, použiť WordPress export/import alebo preniesť konkrétny obsah ručne.

Celowebová migrácia dáva zmysel vtedy, keď sa presúva celý WordPress: téma, pluginy, médiá, nastavenia, menu, formuláre a databáza.

Na čo si dať pozor

Najčastejšie problémy po presune sú staré URL adresy, nefunkčné formuláre, zmiešaný obsah pri HTTPS, cache, chýbajúce obrázky alebo rozdielna verzia PHP na hostingu. Preto presun nekončí importom. Končí až kontrolou webu naživo.

  • prekliknúť menu, služby, blog, kontakt a dôležité vstupné stránky,
  • uložiť permalinky a skontrolovať 404 chyby,
  • otestovať formuláre, SMTP a doručenie emailu,
  • pozrieť obrázky, favicon, logo a mobilné zobrazenie,
  • vyčistiť cache a skontrolovať SSL/mixed content,
  • overiť robots.txt, sitemapu a základné SEO nastavenia.

Ak ide o e-shop, treba navyše otestovať košík, pokladňu, emaily, platby a dopravu. Pri WooCommerce by som migráciu nerobil počas aktívnych objednávok bez jasného plánu.

Krátky záver

Backup Migration vie presun výrazne zjednodušiť, ale stále je to zásah do celého webu. Sprav zálohu, skontroluj hosting, obnov web, vyčisti cache a prejdi najdôležitejšie stránky ručne.

Pri väčšom webe by som migráciu nerobil v štýle „kliknem a uvidíme“. Najprv plán, potom export, import a až nakoniec prepnutie domény alebo DNS.
